# Documento de Especificação do Projeto - Meus Links

## Descrição do Projeto

O projeto "Meus Links" é uma aplicação web frontend que permite aos usuários criar e gerenciar seus próprios links personalizados. Os usuários podem criar uma conta fornecendo um nome de usuário, email e senha, e fazer login no sistema usando essas credenciais. A aplicação irá interagir com a API completa disponível em [https://usuarios.ronierlima.dev/](https://usuarios.ronierlima.dev/) para acessar os dados dos usuários e seus links.

## Objetivos do Projeto

O principal objetivo do projeto "Meus Links" é fornecer aos usuários uma plataforma para criar e gerenciar seus próprios links personalizados. Os objetivos específicos incluem:

- Permitir que os usuários criem uma conta fornecendo um nome de usuário, email e senha, para que possam acessar a aplicação.
- Capacitar os usuários a adicionar, editar e apagar links personalizados, fornecendo um título descritivo, URL, descrição opcional e imagem do link.
- Exibir uma lista de todos os usuários cadastrados, permitindo aos usuários visualizar os detalhes de cada usuário, incluindo sua bio e a lista de links associados.

## Requisitos Funcionais

1. Cadastro de Usuário
    - Os usuários devem poder criar uma conta fornecendo um nome de usuário, email e senha.
    - Os dados de nome de usuário, email e senha devem ser válidos e atender aos critérios de validação.
    - Após o cadastro, os usuários devem poder fazer login no sistema.

2. Login de Usuário
    - Os usuários devem poder fazer login no sistema usando suas credenciais de nome de usuário e senha.
    - As credenciais de login devem ser autenticadas e verificadas pela API.
    - Após o login bem-sucedido, os usuários devem ser redirecionados para a página principal da aplicação.

3. Adicionar Link
    - Os usuários devem poder adicionar um novo link personalizado fornecendo um título descritivo, URL, descrição opcional e imagem do link.
    - Todos os campos obrigatórios devem ser preenchidos para adicionar o link.

4. Editar Link
    - Os usuários devem poder editar um link existente, modificando o título, URL, descrição e imagem.
    - A edição de um link deve ser permitida apenas pelo proprietário do link.
    - Os usuários devem poder salvar as alterações feitas no link.

5. Apagar Link
    - Os usuários devem poder apagar um link existente.
    - A exclusão de um link deve ser permitida apenas pelo proprietário do link.
    - Ao apagar o link, ele deve ser removido da lista de links do usuário.

6. Lista de Usuários
    - A aplicação deve exibir uma lista de todos os usuários cadastrados.
    - Os usuários devem poder visualizar os detalhes de cada usuário ao clicar em seu nome.
    - A lista de usuários deve ser atualizada em tempo real, refletindo as adições e remoções de usuários.

7. Detalhes do Usuário
    - Ao clicar em um usuário na lista, a aplicação deve exibir uma página de detalhes para esse usuário.
    - A página de detalhes deve mostrar a bio do usuário e a lista de links personalizados associados a ele.

## Requisitos Não Funcionais

1. Tecnologias Utilizadas
    - A aplicação deve ser desenvolvida utilizando React, uma biblioteca JavaScript para construção de interfaces de usuário.
    - A biblioteca Axios deve ser utilizada para realizar requisições HTTP à API.
    - A biblioteca React Router Dom deve ser utilizada para gerenciar as rotas da aplicação.

2. Consumo da API
    - A aplicação deve consumir os endpoints da API disponível em [https://usuarios.ronierlima.dev/](https://usuarios.ronierlima.dev/) para interagir com os dados dos usuários e links.
    - A documentação da API está disponível em [https://usuarios.ronierlima.dev/swagger-ui/](https://usuarios.ronierlima.dev/swagger-ui/). Os alunos devem se basear nela para utilizar corretamente os endpoints e parâmetros.

3. Estilização e Componentes
    - Podem ser utilizadas bibliotecas de componentes como Material-UI, Ant Design, React Bootstrap, entre outras, para aumentar a produtividade e consistência da aplicação.
    - Bibliotecas de ícones, fontes e estilos também podem ser utilizadas para melhorar a aparência visual da aplicação.

4. Layout
    - A escolha de cores, fontes e estilos fica a critério da equipe.
